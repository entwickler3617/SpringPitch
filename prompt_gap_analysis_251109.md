## 1) 데이터/입력 관련 모호점 및 결정

- [단위(Units)] 모호 (파일별 상이 가능)
  - IGES Global Section의 단위(inch/mm) 미확인 시 치수가 25.4배 오차 가능.
  - 본 파일(`TK1_FRT_zero-1_251014.igs`) 확인 결과: Global 섹션에 `2HMM` 표기로 mm 단위 사용.
  - 결정: IGES Global 섹션을 읽어 단위를 판별하고, 내부 단위를 mm로 정규화(필요 시 스케일)한다.

- [구분자(Delimiter)] 모호
  - 프롬프트는 단순 정규식으로 `116,x,y,z,0;` 형태를 가정. 실제 IGES는 Global 섹션에 정의된 파라미터/레코드 구분자 사용.
  - 본 파일 확인 결과: `,`(파라미터), `;`(레코드) 구분 패턴 확인됨.
  - 결정: S/G/D/P/T 섹션 파서를 사용해 D→P 매핑으로 116 엔터티를 안전 추출(지수표기 E/D, 공백 포함 허용).

- [변환행렬(Transform 124)] 불확실
  - 일부 파일은 124 엔터티 기반 변환이 필요. 미적용 시 위치 오차.
  - 결정: DE의 Transform Pointer가 있으면 행렬을 적용한 "월드 좌표"로 변환.

- [데이터 품질: 이상치/중복점]
  - 프롬프트는 “곡률 기준 보정”을 요구하나 정량 규칙 부재.
  - 결정(초안):
    - 연속 중복/초근접 점 제거(eps ~ 1e-9~1e-6)
    - 이동평균 스무딩(윈도우 = max(5, n//50)*2+1, 상한 101)
    - 종단/중간의 극단값은 제한적 스무딩만 적용(과보정 금지)

## 2) 좌표계/원점/시작점 관련 모호점 및 결정

- [rθz 원점 정의]
  - 프롬프트: “rθz 좌표계 중심은 스프링 시작점의 중앙” + “직교/원통 동일 원점”.
  - 관찰: 시작점은 “반경(r)이 작은 쪽”으로 정의.
  - 구현 난점: 원시 점들만으로 “정확한 시작점 중앙”은 노이즈/샘플 편차 영향 큼.
  - 결정(권장 절차):
    1) 축은 PCA 주축(Ez) 사용, 프레임(Ex,Ey,Ez) 구성(우수 좌표계 유지)
    2) 축 투영 하위/상위 5% 슬라이스의 반지름 중앙값 비교 → median r가 더 작은 쪽을 시작 슬라이스로 선택
    3) 시작 슬라이스 중심의 축 직투영을 임시 원점(Otemp)으로 설정
    4) 1,000점 표준화 곡선에서 시작 슬라이스 중심에 가장 가까운 표준점 = start_idx
    5) start_idx 기준 θ=0 정규화, θ≈π(180°)에 가장 가까운 반대위상점 opp_idx 산출
    6) “시작·반대위상점의 중점”의 축 직투영을 최종 원점(O)으로 선정

- [직교(xyz)와 원통(rθz) 원점 일치]
  - 프롬프트는 원점 동일화를 요구.
  - 결정: 내부 로컬 좌표(x′,y′,z′) 산출은 모두 O를 기준으로 수행. Excel 표시는 수요에 따라 ‘원시 xyz’ vs ‘로컬 xyz’ 중 선택 필요(명시 요구됨).

- [θ 방향/범위/시작처리]
  - 모호: θ의 증가 방향(우수 좌표계 기준)과 표시 범위.
  - 결정: 우수 프레임에서 arctan2(y′,x′)로 θ(rad) 정의. 분석용은 unwrap, 표시용은 wrap(−π..π) 후 deg(−180..180). 시작 인덱스에서 θ=0 오프셋만 적용(배열 롤링 금지: Z 이음새 아티팩트 방지).

## 3) 표준화(1,000점) 및 보간 규칙

- [표준화 방법]
  - 프롬프트: “전체 topology 유지, 단순 Normalization”만 명시. 정확한 기법 미지시.
  - 결정(기본): Uniform B‑Spline(기본), 대안으로 Linear/NURBS 선택 가능(환경 변수로 전환). 과적합 방지 위해 smoothing 기본 0.0, 필요 시 파라미터화.

- [보간법 적용 규칙]
  - 프롬프트: “해당 위치에 점이 없으면 인접 2점으로 보간”.
  - 결정: 표준화/표시 모두에서 선형 보간 기본 원칙 준수. 스플라인 사용 시에도 최근접 원본 인덱스 매핑 유지.

- [종단/중간 보정]
  - “양끝/중간 일부 튀는 점만 보정” 요청. 정량 기준 미제시.
  - 결정(초안): 이상점 탐지 기준은 이동평균 잔차 z-score>k(예: 3) 등 경량 규칙(옵션)으로 한정, 기본은 최소 보정.

## 4) 치수 정의/계산 관련 모호점 및 결정

- [Turn]
  - θ_unwrapped/(2π)로 계산하되, 시작 인덱스 기준 오프셋 적용.

- [Radius, Diameter(Perpendicular), Diameter(Vectorial)]
  - 정의 불명확(특히 Vectorial). 기본 단계에선 2*R(직교 수직 지름)과 동일 취급, 정밀 차이는 3점 호 기반(step 5)에서 분기.

- [Height]
  - Z 범위(최소~최대)로 정의. 노이즈 대책(평활) 적용 여부 결정 필요.

- [Pitch, min_pitch]
  - Pitch: dZ/dTurn(차분) + 평활화.
  - min_pitch: 구간 최소 피치. 보간/평활 적용 여부, 윈도우 폭 등 명시 필요.
  - 결정(초안): step 4는 차분+평활의 참고값만 제공, step 5에서 3점 호 보간 기반 정밀값 산출.

- [3점 호(Arc) 보간]
  - 프롬프트는 step 5에서 3점 호를 강조. 하지만 구체적 샘플링/구간 폭 미기재.
  - 결정: 표준화 인접점(혹은 가변 간격) 3점을 취해 원 추정 후 반경/중심/호길이로 치수 산정하는 방식을 제안(세부 파라미터는 샘플 시트 수식 확인 필요).

## 9) 2025-11-09 추가 갱신 (Arc / 시작전략 / 평활 / 보고서)

### 신규 구현 및 남은 모호점

- 3점 호(Arc) 기반 정밀 치수:
  - 현재: 로컬 XY 또는 best_fit (PCA 기반 국소평면) 투사 후 3점 원 추정
  - 모호점: Vectorial Diameter 정의가 단순 2R인지, 혹은 축 기울기/비원형 편차를 고려해야 하는지 추가 명세 필요
  - 모호점: Pitch_arc 계산 시 dθ 구간 폭; 현재 2-Point span (i-1,i+1). 확장(가변창) 여부 논의 필요

- Arc 평면 선택 (환경변수: SPRING_ARC_PLANE)
  - 'local_xy' vs 'best_fit' 두 모드 제공
  - best_fit 선택 시 3점 PCA 평면으로 투사 → 약간의 노이즈 제거 장점 있으나 계산 과잉 가능성
  - 결정 필요: 기본 모드 고정 여부(local_xy 유지 제안)

- 시작점 전략 (SPRING_START_STRATEGY)
  - 'min_radius' | 'min_z' | 'max_z' 구현
  - 모호점: 실제 사양서 “반경이 작은 끝” vs 높이(min_z) 우선 순위 충돌 시 정책
  - 제안: 우선 min_radius. 충돌 발생 시 z(min) 보조 판단 규칙 추가 고려

- Seam 사전 보정 (SPRING_SEAM_MODE)
  - min_gap, complete_turns, 조합
  - 모호점: 정수 턴 판정 허용 오차(tol_turn) 기본 0.05 고정 → 환경변수화 필요 여부

- 평활(Smoothing) 모드 (SPRING_SMOOTH_MODE)
  - 'none' | 'robust_ends' | 'robust_all' (설계 중)
  - 현재 moving_average에 robust 옵션 (MAD 기반)만 반영, 전략적 적용 로직 미구현
  - 모호점: robust 적용 범위(끝단 %, z 기울기 급변, 반경 이탈 차이) 수치화 필요

- 최소 피치/국소 피치
  - 현 상태: 차분 기반 + 스무딩, arc 기반 pitch_arc로 대체 가능한 경우 교체
  - 모호점: 최종 보고서에서 사용하는 ‘min_pitch’가 어떤 스무딩/보간 단계를 거친 값인지 명시 필요

- 보고서(report)
  - markdown 형태 초안 작성 (spring_report.md)
  - 이미지/표 출력 세부 구성(결과보고서양식.png 매핑) 추가 필드 명세 필요

### 향후 결정 필요 체크 추가

- Vectorial Diameter: 3D 헬릭스 기울기를 반영해 “수직(축)에 수직한 단면”에서의 외경으로 재정의할지

- 보고서 템플릿: 결과보고서양식.png의 섹션별 필수 항목(제목/단위/소수점 자리) 텍스트 사양화

- 결정: 내부 계산용으로 항상 1,000개 등간격 표준화 수행(기본: 선형 리샘플링). DEFAULT: `SPRING_NORM_METHOD=linear`로 변경.
- 표시는 E,F,G와 분리됨(아래 항목 참조).

• E,F,G(x_norm,y_norm,z_norm) 정의

- 사용자 요구: E,F,G는 원시값을 1:1로 유지하되, “물리적 나선 구조에서 벗어난 이상치만 제한 교정”.
- 구현: `STD_MODE=raw_preserve|raw_with_outlier_fix`에서 E,F,G에 원시좌표(표준화 매핑 순) 기록. `raw_with_outlier_fix` 시 MAD+끝단강화(END_FACTOR)로 선형 보간 교정.
- 보류: “곡률 기반 보정(1.5)”의 정량 규칙은 설명 필요(아래 곡률 항목).

• 표준화 곡선 방식(1.4)

- 결정: 기존 B‑Spline/NURBS는 기본에서 배제. 기본값을 선형으로 변경하여 원시 형상을 최대 보존. 필요 시 환경변수로 전환 가능.

• 시작점/정렬

- 결정: END_SLICE_PERCENT 내 슬라이스 median r 비교로 시작단 선택. `SPRING_ALIGN_START=on`에서 시작 인덱스로 배열 롤(0행=시작점) 적용.
- 설명요망: `coordinate.png` 기준의 회전방향 부호 정의를 문서화(시계/반시계)할지 여부.

• 곡률 기준 보정(1.5)

- 현황: 반경/스텝 MAD 기반 + 끝단 강조(OUTLIER_END_FACTOR, OUTLIER_END_PERCENT). 곡률 proxy(예: 방향벡터 각변화/호장 기반)는 placeholder 상태.
- 설명요망: 곡률 임계(예: 상위 p% or z-score)와 보정 창 폭, 적용 위치(시작/끝 가중) 수치 합의 필요.

• θ 처리 및 나선 진행

- 결정: I열에는 wrapped deg(−180..180)로 표시, Turn은 unwrapped 기반. 시작 정렬 후 양(+) 진행 강제 옵션 유지.
- 설명요망: Opposite phase(π 근접) 선정 시 “반지름 동일” 조건을 얼마나 엄격히 반영할지(현재는 근사).

• 치수 계산과 Excel 열 매핑(S/U/W/X)

- 결정: 표준 시트 수식을 재현해 `zero-1`에 “값만” 기록. 이번 반영 대상: S(벡터 지름), U(Height), W(Pitch), X(min_Pitch).
  - S: R열(arc 보정 지름) 값을 사용해 직접 값 기록(없으면 2R 대체).
  - U: `T - $T$2` 값을 파이썬에서 계산 후 기록.
  - W: `(Z-Z_prev)/(Turn-Turn_prev)`을 파이썬에서 계산해 기록(분모=0은 NA 처리).
  - X: W의 전체 최소를 산출하여 모든 행에 동일 값 기록.
- 보류/설명요망: 표준 시트의 V열 중간항 정의(문구상 “W는 V를 참조”)가 구체 수식 미공개. 현 버전은 V 없이 W를 직접 산출. 원본 수식 제공 시 V도 동등 구현 가능.

• 요약 그리드 Z3:AC14

- 현황: 대표 메트릭(총 높이, 총 권수, 평균/최소 피치 등) 값을 직접 기록. 표준 시트의 정확 공식 매핑은 미확정.
- 설명요망: 각 셀의 기준 공식(평활 여부, 롤링/트리밍 규칙)을 제공받으면 동일 로직으로 재현 가능.

• Outlier/Smoothing 파라미터

- 신규: `SPRING_OUTLIER_END_FACTOR`(기본 0.7), `SPRING_OUTLIER_END_PERCENT`(기본 3.0)로 끝단 강화. 기존 MAD 임계(RADIUS_MAD, STEP_MAD) 병행.
- 계획: 곡률 기반 마스크 도입 시, 끝단 가중과 결합하여 과보정 방지.
