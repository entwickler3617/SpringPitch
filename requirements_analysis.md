# 스프링 역설계 프로그램 개발 - 요구사항 분석 및 검토

## 1. 프로그램 개발 전 명확히 해야 할 사항들

### 1.1 데이터 구조 관련 불일치/모호성

#### 1.1.1 IGS 파일 Point 개수 불일치

- **사양서**: "스프링의 중심선을 1000개로 등분한 포인트데이터"
- **실제 IGS 파일**: 3,141개의 Point 엔티티
- **결정 필요**:
  - 1000개는 이론적 설명인가?
  - 실제 파일의 3,141개를 모두 사용해야 하는가?
  - 1000개로 리샘플링이 필요한가?

#### 1.1.2 좌표계 중심점 정의 모호성

- **사양서**: "rθz 좌표계 중심은 스프링맨위의 중앙"
- **실제 IGS 분석**: 스프링 상단 중심점 (-473.378, 86.256, 449.133)
- **결정 필요**:
  - Z축 원점을 스프링 최상단으로 설정해야 하는가?
  - 아니면 스프링 전체 높이의 기준점을 별도로 설정해야 하는가?

### 1.2 좌표 변환 관련 불명확성

#### 1.2.1 θ(Theta) 계산 기준

- **현재 이해**: θ = arctan2(y, x)
- **불명확 사항**:
  - θ=0도의 기준 방향이 명시되지 않음
  - 시계방향/반시계방향 회전 방향 미정의
  - θ 범위 (0~360도 vs -180~180도)

#### 1.2.2 좌표계 회전/정렬

- **IGS 분석 결과**: 스프링 축이 Z축과 완전히 일치하지 않음
- **결정 필요**:
  - PCA로 구한 주축을 Z축과 정렬해야 하는가?
  - 아니면 원본 좌표계를 그대로 사용해야 하는가?

### 1.3 치수 계산 관련 모호성

#### 1.3.1 Turn 계산 방법

- **사양서 언급**: Turn 값 계산 필요
- **불명확 사항**:
  - 전체 회전수인가? 부분 회전수인가?
  - 360도 기준으로 나눈 값인가?
  - 스프링 길이에 따른 누적 회전각인가?

#### 1.3.2 Diameter 계산의 두 가지 방법

- **Diameter(Perpendicular)**: 수직 방향 직경?
- **Diameter(Vectorial)**: 벡터 방향 직경?
- **결정 필요**:
  - 각각의 정확한 정의와 계산 방법
  - 어느 상황에서 어떤 방법을 사용해야 하는가?

#### 1.3.3 Pitch vs min_pitch

- **Pitch**: 일반적인 피치 (인접 코일 간격)
- **min_pitch**: 최소 피치?
- **불명확 사항**:
  - min_pitch의 정확한 의미
  - 스프링 전체에서 최소값인가? 특정 구간인가?

### 1.4 Interpolation 방법 관련

#### 1.4.1 3점 호(Arc) 보간법

- **사양서**: "3개의 점을 이용한 호(arc)에서 추출"
- **불명확 사항**:
  - 3점 선택 기준 (연속된 점? 특정 간격?)
  - 3D 공간에서의 호 계산 방법
  - 원호의 중심과 반지름 계산 알고리즘

#### 1.4.2 보간 대상 및 목적

- **결정 필요**:
  - 어떤 치수를 위해 보간이 필요한가?
  - 보간 정확도 요구사항
  - 보간 구간 설정 방법

### 1.5 출력 데이터 형식 관련

#### 1.5.1 좌표 변환 테이블

- **출력 필요**: x,y,z,r,θ,z → Turn,Radius,Diameter(Perpendicular),Diameter(Vectorial),Height,Pitch,min_pitch
- **불명확 사항**:
  - 각 포인트별 개별 값인가? 전체 스프링의 대표값인가?
  - 파일 형식 (CSV, Excel, 텍스트?)

#### 1.5.2 Height 계산

- **결정 필요**:
  - 전체 스프링 높이인가?
  - 각 코일별 높이인가?
  - 기준점에서의 상대적 높이인가?

## 2. 현재 확인된 사실들

### 2.1 IGS 파일 분석 결과

- **총 Point 수**: 3,141개
- **좌표 범위**: X(-477.6~-469.1), Y(82.8~89.7), Z(447.1~580.1)
- **스프링 축 방향**: PCA 분석으로 주축 확인 가능
- **중심점**: 계산 가능 (-473.378, 86.256, 449.133)

### 2.2 좌표 변환 가능성

- **xyz → rθz 변환**: 수학적으로 가능
- **r 계산**: √(x² + y²)
- **θ 계산**: arctan2(y, x)
- **z 좌표**: 그대로 사용 또는 기준점 조정

### 2.3 치수 계산 가능성

- **반지름/직경**: r 값에서 통계적 분석 가능
- **피치**: z 좌표 변화율에서 계산 가능
- **높이**: z 좌표 범위에서 계산 가능
